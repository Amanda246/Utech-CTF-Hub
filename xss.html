If this were a real application, you could have stolen cookies, redirected users, or performed actions on their behalf.

Learning Objectives:
  • Understand how XSS vulnerabilities occur
  • Learn the difference between reflected, stored, and DOM-based XSS
  • Recognize unsafe output rendering
How to Prevent XSS:
  • Always HTML-encode user input before displaying it
  • Use templating engines that auto-escape by default
  • Implement Content Security Policy (CSP) headers
  • Validate and sanitize all user input
  • Never use innerHTML with user data — use textContent instead
